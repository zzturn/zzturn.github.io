Maven

Maven 是专门为 Java 项目打造的管理和构建工具，它的主要功能有：

  • 提供了一套标准化的项目结构；
  • 提供了一套标准化的构建流程（编译，测试，打包，发布……）；
  • 提供了一套依赖管理机制。

一个使用 Maven 管理的普通 Java 项目，目录结构默认如下：

a-maven-project
├── pom.xml
├── src
│   ├── main
│   │   ├── java
│   │   └── resources
│   └── test
│       ├── java
│       └── resources
└── target

目录结构都是约定好的标准结构，不要随意修改目录结构。

pom.xml ( Project Object Model)是项目描述文件 

<?xml version="1.0" encoding="UTF-8" ?>
<project xmlns = "http://maven.apache.org/POM/4.0.0"
    xmlns:xsi = "http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation = "http://maven.apache.org/POM/4.0.0
    http://maven.apache.org/xsd/maven-4.0.0.xsd">
 
    <!-- 模型版本 -->
    <!-- 一般都写死了 -->
    <modelVersion>4.0.0</modelVersion>
    
    <!-- groupId 一般是公司或者组织的名字， artifactId 一般是项目名称 -->
    <!-- groupId, artifactId 和 version 共同作为唯一标识 -->
    <!-- 引入第三方库时，也同样如此，用这三个确定所需要的依赖 -->
    <groupId>com.zturn</groupId>
    <artifactId>project</artifactId>
    <version>1.0</version>

    <!-- parent 类似于继承？ 在于复用，在于多模块场景 -->
    <!-- 将公共部分的依赖提取出来，作为 parent，
		 之后在其他模块中就只需要导入 parent就可以了，不用重复配置 -->
    <parent>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-parent</artifactId>
        <version>2.4.4</version>
    </parent>


     <dependencies>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
        </dependency>
    </dependencies>
</project>

依赖

maven 会自动帮我们进行依赖管理，如果我们要导入的包有其他的依赖，maven 会自动帮我们导入 。

依赖关系

scope 说明 示例
compile 编译时需要用到该 jar 包（默认） commons-logging
test 编译 Test 时需要用到该 jar 包 junit
runtime 编译时不需要，但运行时需要用到 mysql
provided 编译时需要用到，但运行时由 JDK 或某个服务器提供 servlet-api 
<dependency>
    <groupId>org.junit.jupiter</groupId>
    <artifactId>junit-jupiter-api</artifactId>
    <version>5.3.2</version>
    <scope>test</scope>
</dependency>

其中 <scope> 标签，用于在项目发布过程中，帮助决定哪些构件被包括进来。

构建流程

Maven 可以自动化实现编译、打包、发布等等。

Lifecycle Phase Goal

Maven 有三套相互独立的的生命周期。

  • clean
  • default
  • site

三套生命周期有各自有其 Phase

如对于 clean 有

  • pre-clean
  • clean
  • post-clean

另外两个 lifecycle 的 phase 就不写了。

当执行 mvn 命令时，后面接的参数就是 phase

常用的 mvn package 就是运行到 default 生命周期的 package 阶段。

常用的命令有：

mvn clean
mvn clean compile
mvn clean test
mvn clean package

执行一个 Phase 会触发多个 Goal

执行的Phase 对应执行的Goal
compile compiler:compile
test compiler:testCompile surefire:test

goal 的命名总是abc:xyz这种形式。

  • lifecycle 相当于 Java 的 package，它包含一个或多个 phase；
  • phase 相当于 Java 的 class，它包含一个或多个 goal；
  • goal 相当于 class 的 method，它其实才是真正干活的。

插件

Maven 执行每个 phase 时，都是通过某个插件来执行的，Maven 本身其实并不知道如何执行compile，它只是负责找到对应的 compiler 插件，然后执行默认的 compiler:compile 这个 goal 来完成编译。

Maven 内置了一些常用的标准插件，但如果标准插件无法满足要求，我们可以使用自定义插件。

自定义插件往往需要一些配置。
